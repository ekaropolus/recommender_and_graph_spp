# Fully Reproducible Scholarship Track

This is the second part of the repository. It is separate from the thesis
archive and implements a deterministic reproducible layer over the committed
thesis materials.

## Purpose

Part I preserves what supports the doctoral thesis. Part II lets a future
reader rerun the computational layer from documented committed inputs to
documented outputs without relying on private notebooks, private database
state, or undocumented manual steps.

## Current Implementation

The reproducible layer is generated by:

```bash
python scripts/build_reproducible_scholarship.py
```

Validate it with:

```bash
python scripts/test_reproducible_scholarship.py
python scripts/validate_repository.py
```

Generated artifacts are committed under `reproducible-scholarship/outputs/`:

- `tables/file_inventory.csv`
- `tables/asdi_datasets.csv`
- `tables/sdg_policy_indicators.csv`
- `graph/nodes.csv`
- `graph/edges.csv`
- `manifests/summary.json`
- `manifests/run-manifest.json`

## Reproducibility Target

A release is fully reproducible when an external reviewer can:

1. identify every input dataset and its source;
2. recreate the software environment;
3. run a deterministic pipeline from source tables to graph artifacts;
4. verify generated outputs using checksums and validation tests;
5. inspect the graph schema and transformation logic;
6. cite a stable release with a DOI, commit hash, and archived artifacts.

## Implemented Scholarly Artifacts

- Data statement for committed source families.
- Machine-readable file inventory with checksums.
- Deterministic script that normalizes committed ASDI and SDG-policy CSVs.
- Graph schema describing node labels, edge types, properties, and constraints.
- Versioned outputs, including node/edge tables and normalized source tables.
- Run manifest for each published execution.
- Tests that validate graph outputs, manifest checksums, and credential hygiene.

## Publication Tasks Still Outside The Repository

- Mint a DOI after archival deposit.
- Add authoritative external source URLs and access dates for every historical
  table where those were not preserved in the original thesis archive.
- Add Cypher migrations if the release must recreate a Neo4j database rather
  than use graph CSV artifacts.
- Update `CITATION.cff` with formal release metadata after DOI minting.

## Proposed Future Layout

The current thesis archive keeps its historical folder names. The reproducible
track now adds deterministic outputs while leaving historical paths intact:

```text
reproducible-scholarship/
  data-statement.md
  graph-schema.md
  methodology.md
  graph/
  outputs/
    graph/
    manifests/
    tables/
```

Do not move the thesis archive until compatibility with existing notebook links
has been handled.

## Boundary From The Thesis Archive

This track is not additional evidence that the 2022 thesis already contained a
finished reproducible software package. It is a scholarly hardening path built
from the thesis materials. The distinction matters for accurate citation and
review.
